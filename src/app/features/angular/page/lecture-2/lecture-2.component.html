ლექცია 2 – კომპონენტები, Template და Data Binding

რა არის კომპონენტი?

Angular აპლიკაციის მთავარი „სამშენებლო ბლოკი“ არის კომპონენტი. თითოეული კომპონენტი პასუხს აგებს UI-ის კონკრეტულ ნაწილზე და შეიცავს როგორც ლოგიკას, ასევე ვიზუალურ ნაწილს. ვებ-გვერდის თითოეული გვერდი ეწყობა სხვადასხვა კომპონენტების კომბინაციით.

კომპონენტი შედგება 3 ძირითადი ნაწილისგან:

  • TypeScript ფაილი – კომპონენტის ლოგიკა, მონაცემები და ქცევა.
  • HTML Template – ის, თუ როგორ გამოიყურება კომპონენტი ეკრანზე.
  • CSS/SCSS – სტილები, რომლებიც განსაზღვრავს კომპონენტის დიზაინს.

მთლიანობაში Angular აპლიკაცია წარმოადგენს ერთმანეთთან დაკავშირებული კომპონენტების ერთობლიობას. ქვემოთ მოცემულ ფოტოზე ნაჩვენებია, თუ როგორ არის Angular-ის ოფიციალური დოკუმენტაციის ერთი გვერდი დაყოფილი ცალკეულ კომპონენტებად.

Data Binding-ის ტიპები

Data Binding უზრუნველყოფს კავშირს TypeScript და HTML Template-ს შორის. Angular-ში არსებობს ოთხი ძირითადი ტიპი:

1. Interpolation ({{ }})

გამოიყენება მონაცემების გამოსატანად HTML-ში.

HTML:

<p>{{ title }}</p>

TypeScript:

title = 'Angular Course';

2. Property Binding ([ ])

გამოიყენება HTML ელემენტის ატრიბუტებზე მნიშვნელობის გადასაცემად.

HTML:

<img [src]="imageUrl" />

TypeScript:

imageUrl = 'assets/logo.png';

3. Event Binding (( ))

გამოიყენება მოვლენებზე რეაგირებისთვის.

HTML:

<button (click)="onClick()">Click me</button>

TypeScript:

onClick() {
  console.log('Button clicked');
}

4. Two-Way Binding ([( )])

მონაცემების ორმხრივი კავშირი UI-სა და ლოგიკას შორის.

HTML:

<input [(ngModel)]="name" />
<p>Hello {{ name }}</p>

TypeScript:

name = '';

კომპონენტის შექმნა

როგორც აღვნიშნეთ პროექტის შექმნისას ავტომატურად იქმნება მთავარი კომპონენტი app.component, თუმცა ჩვენ შეგვიძლია შექმნათ სხვა კომპონენტებიც. კომპონენტის შესაქმნელად არსებობს რამდენიმე საშუალება.
1. Angular cli-ის საშუალებით

Angular cli-ის საშუალებით შესაძლებელია კომპონენტის შექმნა ბრძანებით

ng generate component name-of-the-component
2. ედიტორში დავამატოთ extension, მაგალითად vsCode-ისთვის Angular Files

შემდეგ იმ ფოლდერზე სადაც გვინდა რომ კომპონენტი შევქმნათ დავაკლლიკებთ მარჯვენა ღილაკს და ავირჩევთ generate-component-ს, შემდეგ კი ჩავწერთ კომპონენტის სახელს, რაც გვინდა რომ ერქვას კომპონენტს

3. ხელით შევქმნათ ყველა საჭირო ფაილი: ts,css,html

    CourseComponent
  ],
  imports: [
    BrowserModule,
    FormsModule
  ],

3️⃣ course.component.ts (Class + Component დეკორატორი)

import { Component } from '@angular/core';

@Component({
  selector: 'app-course',
  templateUrl: './course.component.html',
  styleUrls: ['./course.component.scss']
})
export class CourseComponent {
  // Data Binding - ცვლადები
  title: string = 'Angular Courses';
  description: string = 'ეს არის პირველი პრაქტიკული კომპონენტი';

  // Two-way Binding ცვლადი
  inputValue: string = '';

  // Method - Event Binding
  showAlert() {
    alert('კომპონენტის Title: ' + this.title);
  }
}

როგორც აღვნიშნეთ, Angular-ში UI მთლიანად აგებულია კომპონენტებისგან. თითოეული კომპონენტი წარმოადგენს UI-ის ერთ დამოუკიდებელ ნაწილს. მაგალითად: Header, Footer, Product Item, User Card და სხვა.

კომპონენტების იერარქია

წარმოვიდგინოთ, რომ გვაქვს app კომპონენტი, რომელშიც გამოძახებული გვაქვს სხვა კომპონენტები, მათ შორის header და footer.

app კომპონენტს ვუწოდოთ მშობელი კომპონენტი, ხოლო header და footer კომპონენტებს შვილი კომპონენტები. კომპონენტების იერარქია უნდა წარმოვიდგინოთ ხის სახით და მარტივი გასაგებია მშობელი-შვილი სტრუქტურა.

რატომ გვჭირდება კომუნიკაცია?

მაგალითად, app კომპონენტში გვაქვს name = 'John Doe' და გვინდა, რომ ეს name გამოვიტანოთ header კომპონენტში.

კომპონენტში აღწერილი ცვლადის მნიშვნელობა მხოლოდ იგივე კომპონენტის HTML-მა იცის. ე.ი. თუ app კომპონენტში გვიწერია name, ნიშნავს რომ header-ის HTML-ში პირდაპირ წვდომადი არ იქნება. სწორედ ამისთვის არსებობს კომპონენტების შორის კომუნიკაცია.

ინფორმაციის გადაცემის მიმართულება

Angular აპლიკაცია არის კომპონენტების ხე. ინფორმაციის გაცვლისას:

  • მონაცემები გადადის ზემოდან ქვემოთ (მშობელი კომპონენტიდან → შვილ კომპონენტზე)
  • ივენთები ბრუნდება ქვემოდან ზემოთ (შვილი კომპონენტიდან → მშობელ კომპონენტზე)

კომპონენტების შორის კომუნიკაციის ტიპები

1. @Input() - მონაცემების გადაცემა

როდესაც მშობლიდან მონაცემს გადავცემთ შვილ კომპონენტს.

@Input() სინტაქსი:
  • @Input() propertyName: type = defaultValue; - ძირითადი სინტაქსი
  • @Input() product!: Product; - Non-null assertion (!) - დარწმუნებული ხართ რომ მნიშვნელობა იქნება
  • @Input() optionalData?: string; - Optional (?) - მნიშვნელობა შეიძლება იყოს undefined
  • @Input('externalName') internalName: string; - Custom property name - გარედან სხვა სახელით გადაცემა

💻 ინტერაქციული რედაქტორი - პრაქტიკული მაგალითი

ქვემოთ შეგიძლიათ ნახოთ @Input()-ის მუშაობა:

კონტროლები:

სცადეთ თავად შეუცვალოთ მნიშვნელობა Parent Component-ში და დააკვირდით როგორ იცვლება Child Component-ში
Parent Component (app.component)

name = "{{ inputDemoName }}"

↓ გადაცემულია Child Component-ში

2. @Output() - ივენთების გადაცემა

როდესაც შვილი კომპონენტიდან მოქმედებას (ივენთს) გადავცემთ მშობელ კომპონენტს.

@Output() სინტაქსი:
  • @Output() eventName = new EventEmitter<Type>(); - ძირითადი სინტაქსი
  • @Output('customEvent') internalEvent = new EventEmitter<string>(); - Custom event name
  • @Output() dataEmitted = new EventEmitter<Product>(); - Complex data type
  • eventName.emit(data); - ივენთის გაგზავნა
  • (eventName)="handler($event)" - Parent-ში მიღება
