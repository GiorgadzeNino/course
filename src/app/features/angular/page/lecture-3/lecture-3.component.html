Module 3: Angular Directives & Pipes

1. რა არის Directive Angular-ში

Directive არის მექანიზმი, რომელიც ცვლის HTML ელემენტის ქცევას ან გარეგნობას.

კომპონენტიც სინამდვილეში არის დირექტივა, მაგრამ:

  • კომპონენტს აქვს შაბლონი (template)
  • დირექტივას — არა

მარტივად: დირექტივა = მოქმედების დამატება HTML-ის ელემენტებზე

2. Directives-ის ტიპები

Angular-ში გვაქვს 3 ძირითადი ტიპი:

  • Component Directives – UI კომპონენტები (Template + Class + Styles)
  • Structural Directives – ელემენტების შექმნა/წაშლა DOM-ში (*ngIf, *ngFor)
  • Attribute Directives – ელემენტის ატრიბუტების, სტილის ან ქცევის შეცვლა ([ngClass], [ngStyle])

3. Structural Directives (სტრუქტურული დირექტივები)

Structural directive:

  • ცვლის DOM-ის სტრუქტურას
  • ამატებს ან შლის ელემენტებს

შენიშვნა: ქვემოთ მოცემული გვაქვს მაგალითები ორი განსხვავებული ვერსიით როგორ იწერებოდა ძველ ვერსიებში და როგორ ახალში. თუმცა უნდა აღვნიშნოთ რომ თუ ახალ ვერსიაშიც გვეეწერება ძველი სინტაქსით, კოდი იმუშავებს ჩვეულებრივად და არ ექნება პრობლემა

🔑 ყოველთვის იწყება *-ით

3.1 *ngIf

<p *ngIf="isLoggedIn">Welcome!</p>
  • ე.ი. გვაქვს ცვლადი isLoggedIn და თუ isLoggedIn === true → ელემენტი ჩანს
  • თუ მას მნიშვნელობა არ აქვს ან მისი მნიშვნელობააfalse → ელემენტი საერთოდ არ არსებობს DOM-ში

3.2 *ngFor

<li *ngFor="let item of items">
    {{ item }}
</li>

გამოიყენება სიების დასარენდერებლად. თითოეული ელემენტი თავისი მნიშვნელობით ერთიანად გამოდის DOM-ში როგორც <li> ელემენტი.

როდესაც *ngFor-ს არ უწერია trackBy ფუნქცია, როდესაც ლისტი განახლდება, ანგულარი დომიდან წაშლის ყველა ელემენტს და ახლიდან დახატავს, როდესაც ბევრი მონაცემი გვაქვს, დამეთანხმებით, პერფორმასისთვის ცუდია ერთი მონაცემის ცვლილების გამო ყველას თავიდან გადაწერა

trackBy ფუნქციით კი თითოეულ ელემენტს ვანიჭებთ რაღაც უნიკალურ მნიშნელობას, რომელიც უშუალოდ განახლებულ ან დამატებულ ელემენტს შეცვლის დომში და არ გადახატავს ახლიდან მთლიან ლისტს

3.3 *ngSwitch

<div [ngSwitch]="role">
    <p *ngSwitchCase="'admin'">Admin</p>
    <p *ngSwitchCase="'user'">User</p>
    <p *ngSwitchDefault>Guest</p>
    </div>

ℹ️ ახალი სინტაქსი: Angular-ის ბოლო ვერსიებში შეგვიძლია template-ში გამოვიყენოთ @if, @for და @switch სტრუქტურული დირექტივებისთვის.

🔑 3.1 @if

@if (isLoggedIn) {
  <p>Welcome!</p>
} @else {
  <p>Please log in</p>
}
  • isLoggedIn არის ცვლადი.
  • თუ isLoggedIn === true → DOM-ში ჩანს <p>Welcome!</p>.
  • თუ მნიშვნელობა არის false ან undefined → მუშაობს @else ბლოკი და გაეშვება <p>Please log in</p>.

🔑 3.2 @for

<ul>
  @for (item of items; track item) {
    <li>{{ item }}</li>
  }
</ul>
  • items არის სია/მონაცემთა მასივი (მაგალითად: ['Angular', 'React', 'Vue']).
  • თითოეული ელემენტი თავისი მნიშვნელობით ერთიანად გამოდის DOM-ში როგორც <li> ელემენტი.
  • track item ეხმარება Angular-ს ეფექტურად განაახლოს სია.

🔑 3.3 @switch

@switch (role) {
  @case ('admin') { <p>Admin</p> }
  @case ('user')  { <p>User</p> }
  @default        { <p>Guest</p> }
}
  • role არის ცვლადი (მაგ. 'admin', 'user' ან 'guest').
  • DOM-ში მხოლოდ ის <p> ელემენტი ჩნდება, რომელიც ემთხვევა მიმდინარე მნიშვნელობას.
  • თუ არც ერთი @case არ დაემთხვა → მუშაობს @default ბლოკი.

trackBy vs track - შედარება

ფუნქცია გამოიყენება მიზანი სავალდებულო?
trackBy *ngFor ფუნქცია, რომელიც აბრუნებს უნიკალურ იდენტიფიკატორს თითოეული ელემენტისთვის, DOM-ის ეფექტური განახლებისთვის. არასავალდებულო (მაგრამ ძალიან რეკომენდირებულია პროდუქტიულობისთვის)
track @for გამოსახულება, რომელიც აბრუნებს უნიკალურ იდენტიფიკატორს თითოეული ელემენტისთვის, სწრაფი diff-ისთვის. სავალდებულო ახალი @for-ის შემთხვევაში სწრაფი DOM განახლებების უზრუნველყოფისთვის
trackBy - *ngFor

გამოიყენება: *ngFor

მიზანი: ფუნქცია, რომელიც აბრუნებს უნიკალურ იდენტიფიკატორს თითოეული ელემენტისთვის, DOM-ის ეფექტური განახლებისთვის.

სავალდებულო? არასავალდებულო (მაგრამ ძალიან რეკომენდირებულია პროდუქტიულობისთვის)

track - @for

გამოიყენება: @for

მიზანი: გამოსახულება, რომელიც აბრუნებს უნიკალურ იდენტიფიკატორს თითოეული ელემენტისთვის, სწრაფი diff-ისთვის.

სავალდებულო? სავალდებულო ახალი @for-ის შემთხვევაში სწრაფი DOM განახლებების უზრუნველყოფისთვის

4. Attribute Directives (ატრიბუტული დირექტივები)

11. რატომ შეცვალა Angular-მა *ngIf / *ngFor?

Angular-ის ახალ ვერსიებში შემოღებულია ახალი სინტაქსი @if, @for და @switch ძველი *ngIf, *ngFor და *ngSwitch-ის ნაცვლად. ქვემოთ მოცემულია მიზეზები, რატომ გაკეთდა ეს ცვლილება:

მიზეზი ახსნა
შესრულება ნაკლები DOM ოპერაციები
წაკითხვადობა უფრო ჰგავს JS/TS-ს
კომპილაცია უკეთესი tree-shaking
შესრულება

მიზეზი: ნაკლები DOM ოპერაციები

ახალი სინტაქსი უფრო ეფექტურად მუშაობს DOM-თან, რაც იწვევს უკეთეს პერფორმანსს.

წაკითხვადობა

მიზეზი: უფრო ჰგავს JS/TS-ს

ახალი სინტაქსი უფრო ბუნებრივად იკითხება და ჰგავს JavaScript/TypeScript კოდს.

პროგნოზირებადობა

მიზეზი: აღარ არის * მაგია

ძველი * სინტაქსი იყო "მაგიური" - Angular-მა იცოდა რომ ეს იყო დირექტივა, მაგრამ ახალი სინტაქსი უფრო გამოკვეთილი და გასაგებია.

კომპილაცია

მიზეზი: უკეთესი tree-shaking

ახალი სინტაქსი საშუალებას აძლევს კომპილატორს უკეთესად გაიგოს რა კოდი გამოიყენება და რა არა, რაც იწვევს უფრო პატარა bundle-ებს.

6. რა არის Pipe Angular-ში

Pipe არის მექანიზმი, რომელიც გამოიყენება მონაცემების ტრანსფორმაციისთვის template-ში.

ანუ pipe საშუალებას გვაძლევს:

  • მონაცემი UI-ში წარმოვაჩინოთ სხვა ფორმატში
  • ორიგინალი მონაცემი არ შეიცვალოს
  • ტრანსფორმაცია მოხდეს template level-ზე

მაგალითი:

<p>{{ user.name | uppercase }}</p>

Custom Pipes & Pure vs Impure (მოკლე იდეა)

ხშირად რეალურ პროექტში გვჭირდება საკუთარი pipe (მაგ. truncate, mask phone). Pipe-ები default არის pure და მუშაობს მხოლოდ მაშინ, როცა input reference შეიცვალა.

თუ impure pipe-ს დავნიშნავთ, ის იმუშავებს ყველა change detection-ზე და შეიძლება performance დაგვიზიანოს.
