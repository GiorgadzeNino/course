ლექცია 1

1. Angular მიმოხილვა

Angular არის თანამედროვე Front-End ფრეიმვორკი, რომელიც შექმნილია Google-ის მიერ. ეს ფრეიმვორკი საშუალებას გვაძლევს შევქმნათ მაღალი ხარისხის, სწრაფი და რეაქტიული ვებ-აპლიკაციები.

რატომ Angular?

  • აქტიური მხარდაჭერა Google-ისგან
  • მარტივი მასშტაბირება პატარა და დიდ პროექტებში
  • მდგრადი ეკოსისტემა და ფართო საზოგადოება
  • მრავალჯერადი გამოყენების კომპონენტები და მოდულები

მოკლე მაგალითი: Angular-ის გამოყენებით მარტივად შეგვიძლია შექმნათ "Hello World" აპლიკაცია, რომელიც გამოჩნდება ჩვენს ბრაუზერში.

Angular-ის ინსტალაცია

  1. შევამოწმოთ გვაქვს თუ არა Node.js დაინსტალირებული, შემდეგი ბრძანებით: 
    node -v
    თუ დაინსტალირებული გაქვთ უკვე, გამოგვიჩენს შესაბამის ვერსიას. თუ ჯერ არ გვაქვს დაინსტალირებული, დააინსტალირეთ Node.js.
  2. როდესაც დავაინსტალირებთ Node.js-ს, ავტომატურად უნდა დააინსტალიროს Node's package manager NPM. იმისთვის რომ შევამოწმოთ დაინსტალირებულია თუ არა npm, ტერმინალში გავუშვათ ბრძანება: 
    npm -v
  3. შემდეგ უნდა დავაინსტალიროთ უშუალოდ Angular CLI.
    Angular CLI არის ერთ-ერთი ყველაზე ძლიერი ხელსაწყო Angular-ში, რომლითაც ის აღემატება სხვა Front-End ფრეიმვორკებს.
    დასაინსტალირებლად გავუშვათ შემდეგი ბრძანება: 
    npm install -g @angular/cli
    ინსტალაციის შემდეგ შეგვიძლია შევამოწმოთ CLI-ის ვერსია, რათა დავრწმუნდეთ რომ ის დაინსტალირდა: 
    ng v

CLI-ის ძირითადი შესაძლებლობები

3.1 აპლიკაციების სწრაფი შექმნა

ng new my-app

შექმნის ახალ Angular პროექტს საჭირო სტრუქტურით, კონფიგურაციით და დაწყებით ფაილებით.

3.2 კომპონენტების, სერვისების და მოდულების გენერირება

ng generate component my-component
ng generate service my-service

ეს გაძლევთ საშუალებას მარტივად დაამატოთ ახალი ელემენტები პროექტში, ავტომატურად შექმნილი საჭირო ფაილებით და კოდით.

3.3 აპლიკაციის კონფიგურაცია და მოდიფიკაცია

CLI საშუალებას გვაძლევს შეცვალოთ კონფიგურაცია, ჩავრთოთ/გამოვრთოთ ფუნქციები და მოვახდინოთ ავტომატური ოპტიმიზაციები.

ნელ-ნელა კურსის განმავლობაში უკეთესად გავეცნობით CLI-ს შესაძლებლობებს. ახლა კი დავიწყოთ აპლიკაციის შექმნით.

შენიშვნა: Angular CLI-ს შესაძლებლობები გაცილებით მეტია, ვიდრე აქ ნახსენები ოპერაციები. პირველ ეტაპზე კი ყველაზე ხშირად სწორედ ეს ძირითადი ფუნქციები გჭირდებათ.

ახალი აპლიკაციის შექმნა

ng new my-app არის Angular CLI ბრძანება, რომელიც ქმნის ახალ Angular პროექტს. აქ ყველაფერი ხდება ავტომატურად, რათა თქვენ შეძლოთ სწრაფად დაიწყოთ მუშაობა.

ნაწილი აღწერა
ng ეს არის Angular CLI-ს მთავარი ბრძანება. მის გარეშე Angular ბრძანებები არ იმუშავებს.
new ნიშნავს, რომ CLI ახალ Angular პროექტს შექმნის.
my-app ეს არის თქვენი ახალი აპლიკაციის სახელი. შეგიძლიათ ნებისმიერი სახელი შეარჩიოთ (მაგ. todo-app, shop-dashboard).

რას აკეთებს CLI, როდესაც ng new ბრძანება გაეშვება

  1. CLI შექმნის ახალ ფოლდერს სახელით my-app (ან სხვა თქვენ მიერ არჩეული).
  2. Angular პროექტის სტრუქტურის გენერირება.
  3. კონფიგურაციის პარამეტრების არჩევა – ბრძანების შესრულების დროს CLI გკითხავთ, მაგალითად რომელი სტილის პრეპროცესორი გინდა გამოიყენოთ. სასურველი პრეპროცესორის ასარჩევად შეგვიძლია ისრებით ვიმოძრაოთ ტერმინალში და დავაკლიკოთ Enter-ს.
  4. გვინდა თუ არა SSR? ეს ტექნოლოგიებია, რომლებიც საშუალებას აძლევენ Angular აპლიკაციას სერვერზე წინასწარ გენერირებული HTML შესთავაზოს, რაც აჩქარებს პირველ რენდერს და აუმჯობესებს SEO-ს. პირველ ეტაპზე, სწავლების და პატარა აპლიკაციების შექმნის დროს, ჩვენ ეს ფუნქცია არ გვჭირდება, ამიტომ ჩვეულებრივ ამ კითხვაზე პასუხი არის No.

პროექტის ძირითადი ფაილები

  • angular.json – აპლიკაციის კონფიგურაციის ფაილი. აქ განისაზღვრება პროექტის სტრუქტურა, ბილდები და სხვ.
  • node_modules/ – ყველა საჭირო ბიბლიოთეკის კოდის კოლექცია, რომ Angular აპლიკაციამ იმუშაოს.
  • package.json – ინფორმაცია პროექტისთვის აუცილებელ პაკეტებზე და მათ ვერსიებზე.
  • package-lock.json – დეტალური მონაცემები პაკეტებს შორის დამოკიდებულებებისა და ვერსიების შესახებ.
  • README.md – ფაილი, სადაც შეგვიძლია აღწეროთ პროექტი, ინსტრუქციები ან სხვა ინფორმაცია სხვებისთვის თვალსაჩინოდ.
  • src/ – ძირითადი ფოლდერი, სადაც აპლიკაციის ყველა კოდი და რესურსი არის განთავსებული.
  • app/ – მთავარი აპლიკაციის მოდულების და კომპონენტების ფოლდერი:
    • app.component.ts – მთავარი კომპონენტის ლოგიკა
    • app.component.html – მთავარი კომპონენტის მარკაპი
    • app.component.css – მთავარი კომპონენტის სტილი
    • app.component.spec.ts – კომპონენტის ავტომატიზირებული ტესტ ფაილი
    • app.config.ts – აპლიკაციის კონფიგურაციის კოდი
    • app.routes.ts – აპლიკაციის რუტინგის კონფიგურაცია
  • assets/ – ყველა რესურსი, როგორიცაა სურათები, აიქონები, JSON ფაილები და სხვა.
  • favicon.ico – აპლიკაციის აიქონი, რომელიც ბრაუზერის ტაბზე ჩანს.
  • index.html – აპლიკაციის მთავარი ფაილი, რომელიც ჩართავს მთელ აპლიკაციას.
  • main.ts – ამ ფაილის საშუალებით ინიციალიზდება Angular.
  • styles.css – გლობალური სტილების ფაილი, რომელიც მოქმედებს ყველა კომპონენტზე.
  • tsconfig.json – ძირითადი TypeScript კონფიგურაცია.
  • tsconfig.app.json – TypeScript კონფიგურაცია Angular კომპილერისთვის.
  • tsconfig.spec.json – TypeScript კონფიგურაცია ტესტების ფაილებისთვის.

4. დაინსტალირება და გაშვება

CLI ავტომატურად ასრულებს შემდეგ ბრძანებას, რათა დაინსტალირდეს ყველა საჭირო ბიბლიოთეკა Angular-ისთვის და შესაბამისად დაემატება ყველაფერი node_modules ფოლდერში:

npm install

პროექტის შექმნის შემდეგ აპლიკაციის გასაშვებად ვასრულებთ:

npm run start

ან

ng serve

ან ბრაუზერის ავტომატურად გასახსნელად:

ng serve --open

ბრაუზერში ვხსნით:

http://localhost:4200

სადაც შეგვიძლია ვიხილოთ ჩვენ მიერ შექმნილი Angular აპლიკაცია.

ანგულარი ნაგულისხმევად ისტარტება 4200 პორტზე, თუმცა ჩვენ შეგვიძლია შევუცვალოთ პორტი და პროექტი დავსტარტოთ მაგალითად 4201.
ng serve --port 4201 ბრძანებით
