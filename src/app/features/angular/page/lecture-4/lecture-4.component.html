Module 4: Angular Services

1. რა არის Service Angular-ში

Serviceარის სპეციალური TypeScript კლასი, რომელიც გამოიყენება ლოგიკის, მონაცემების ან განმეორებადი ფუნქციონალის კომპონენტებში გამოყენებისთვის.

შედარებისთვის

  • Component – პასუხისმგებელია UI-სთვის, template-ისა და view-ის მართვისთვის.
  • Service – პასუხისმგებელია ლოგიკის, მონაცემთა მართვის, API-ქოლების ან საერთო ფუნქციების შესანახად.

2. Service-ის შექმნა

CLI ბრძანება:

ng generate service my-service

ან მოკლე ფორმა:

ng g s my-service

ეს შექმნის ორ ფაილს:

  • my-service.service.ts – სერვისის კლასი
  • my-service.service.spec.ts – სერვისის ტესტირების ფაილი(unit test)

მაგალითი:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class MyServiceService {

  constructor() { }

  greet(name: string): string {
    return `გამარჯობა, ${name}!`;
  }
}

შენიშვნა:@Injectable({ providedIn: 'root' }) – ნიშნავს, რომ service singleton-ია მთელ აპლიკაციაში, ანუ ყველგან ერთსა და იმავე instance-ს ვიყენებთ. მარტივად რომ ვთქვათ 👇 Service იქმნება მხოლოდ ერთხელ. შემდეგ ამ ერთი და იმავე სერვისს იზიარებს ყველა კოპმონენტი. ანუ აპლიკაციაში არ იქმნება მისი ბევრი ასლი.

3. Service-ის მაგალითები რეალურ პროექტში

  • API Service - backend request-ები (HttpClient)
  • Auth Service - login/logout, token handling
  • Storage Service - localStorage/sessionStorage abstraction
  • Notification Service - toast/snackbar შეტყობინებები
  • Feature Service - domain-logic (მაგ. CartService, OrdersService)

4.1 HTTP მონაცემების მიღება API-დან

Service და Component ერთ რედაქტორში (ფაილის არჩევით):

  • {{ user.name }}

დამატებითი მაგალითი (greet service)

  • private userService: UserService -> dependency injection.
  • HTTP მონაცემი მოდის this.userService.getUsers()-ით.
  • Preview-ში ნაჩვენებია API-დან დაბრუნებული user list-ის მაკეტი.

4. Sharing Data Between Components

Service-ში მონაცემის შენახვა

User: {{ sharedPreviewUserName || '---' }}

აქ გვაქვს:

  • userName -> საერთო მონაცემი
  • setUserName() -> მონაცემის შეცვლა
  • getUserName() -> მონაცემის მიღება

3. Component A — მონაცემის გაგზავნა

აქ component აგზავნის მონაცემს service-ში.

4. Component B — მონაცემის მიღება

აქ მეორე component იღებს მონაცემს.

5. Best Practices

  • Services არ უნდა ემყარებოდეს DOM-ს.
  • Singleton pattern – გამოიყენე providedIn: 'root', თუ სერვისის ლოგიკა გამოიყენება მთელ აპლიკაციაში.
  • Modular services – მოდულების მიხედვით შეადგინე service-ები, რომ bundle size არ გაიზარდოს.
